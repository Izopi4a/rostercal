rostercal — examples

A small calendar library. Month view + Resource Time Grid. MIT.

  • month-basic M2

    Month grid with a handful of fixture events.

  • month-themes M2

    Same month view, swap between light, dark, and high-contrast themes at runtime.

  • imperative-api M3

    Add/remove events and navigate via the public API. Shows listener payloads in a log.

  • dnd M4

    Drag events between days; drag the right edge to resize; or use the keyboard. Toggle a rejection handler to see optimistic rollback.

  • rtg-worker-scheduling M5

    Resource Time Grid view: workers as columns, 30-min slots, drag appointments between workers and times.

  • rtg-many-resources M5

    Eight columns with overlapping events to show column splitting (50/50, thirds, etc.).

  • rtg-working-hours v0.3.0

    Working window 08:00–18:00, lunch breaks per worker rendered as blocked ranges, price chip via eventContent. Toggle allowDropOnBlocked to see dropRejected fire (or not).

  • crud-url-shorthand M6

    Wire each operation to "METHOD /path/:id". A localStorage-backed fake fetcher serves the requests; toggle a "simulate 500" switch to see optimistic rollback + dataError.

  • crud-function-form M6

    Each operation is a plain function. fromServer/toServer translate a backend DTO shape (uuid/name/at/untilAt) to/from RosterEvent.

  • nail-salon M7

    Fullstack salon booking app. Add workers and services, then drag a service card onto the Resource Time Grid to create appointments. Backed by a real PostgreSQL database via a TypeScript (Hono) server — Go and PHP backends coming soon.
